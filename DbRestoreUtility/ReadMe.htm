DB Restore Utility

This utility is created only for terakora employees So that users can archive, restore & delete DB and save installation time. Users are not allowed to share it outside.
This utility is in developing mode. So please use it carefully.

  1. This utility will create one backup folder contains backup file & other sql files. The directory user selects to use it as backup directory must be empty.
  2. Backup directory contains one Database back info. Properties file. Which will give fair idea to user about which db is archived, when it is archived, DB IP etc.
  3. This utility will archive & restore all the DB Present (along with children db) in DB Topology. No need to specify Children db separately.
  4. Lock Release will release the locks acquired due to abnormal termination of archiving or restoration of db.
  5. DB Delete will delete all objects present in database along with its children database objects. Use Fast delete option carefully because it will lock other users from accessing database.

Requirements for using this software

  1. Arcmain is installed on your system. To test type arcmain in cmd.
  2. Bteq is installed on your system.
  3. Java 6 is installed on your system & Java environment variables are properly set.

What You Most Need to Know About DB Restore Utility

  1. Please Close all DB Connections & server for the DB you are going to operate on.
  2. Never close window during archiving, restoration & Deletion operation. It may harm your db.
  3. Never temper the backup files & other files created by software.
  4. Do not start two operations simultaneously.
  5. You may see error code 8 in restoration log due to some backup tables not present. That's ok. Known issue
  6. Return Code 4 & 8 is for warnings, but the normal flow of the execution will continue.


Naming of Backup Folder

  • DB_Version = specifies MDM version.
  • _MT = If it’s multitable.
  • _CAPP = If it’s Custom_app along with ISG done.
  • _DATA = If test data has been added in instance.
  • _Linux = If it is linux environment.
  • _ISG = If ISG is done.
  • _UISG = If ISG is done from upgraded version.

Issues Fixed

  • Primary key issue for temporal tables
  • Index issues while restoring.
  • Index issues while Copying.
  • Db Locking during deletion.
Let say I’m archiving 3101 with custom app in Linux. Then folder Name will be – “DB_3101_CAPP_LINUX”
